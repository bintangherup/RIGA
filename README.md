# RIGA — Hyperliquid Trading Dashboard

A self-hosted Streamlit dashboard that helps a **discretionary trader** screen news, watch live markets, find consolidation setups, and size positions for trades on [Hyperliquid](https://app.hyperliquid.xyz). Uses only public APIs — **no auto-execution, no recommendation engine.** You decide; the dashboard does the math and synthesis.

Designed to be used alongside **[Claude Code](https://claude.com/claude-code)** — the AI screening summary is generated by Claude reading a JSON file the dashboard exports, and writing a markdown summary the dashboard reads back.

> ⚠️ **Not financial advice.** This tool is for personal research and decision support. You are responsible for every trade you place.

---

## What it does

The dashboard mirrors a 3-layer discretionary trading workflow:

1. **Layer 1 — Screening:** what's interesting today across the Hyperliquid universe and the broader macro landscape
2. **Layer 2 — Deep dive:** for a candidate asset, what's the catalyst (news, structural shifts)
3. **Layer 3 — Entry:** signals + position sizing for a specific trade idea

### Panels (top to bottom of the dashboard)

| # | Panel | Refresh | What it shows |
|---|---|---|---|
| 1 | **Top Movers** | 5s | Top 5 gainers / losers / over-longed / over-shorted / volume leaders across all Hyperliquid main-DEX perps (vol24h ≥ $1M filter). Ticker-agnostic momentum discovery. |
| 2 | **Consolidation Scanner** | 10min | Composite score (BB-width percentile + 24-bar range/price + centeredness) on 1h candles. Ranks all perps; top 5 rendered as candlestick charts with the consolidation zone highlighted. Identifies breakout-watch candidates. |
| 3 | **Per-asset Hyperliquid live** | 3s | Mark, funding rate, open interest, top-10 order book + imbalance for your watchlist. **Multi-DEX-aware** (handles `HYPE` and `xyz:BRENTOIL`). |
| 4 | **TradingView Chart** | live (iframe) | Embedded TradingView widget with MACD + VWAP enabled by default. Configurable asset, timeframe, venue, theme. |
| 5 | **Trade Decision (Layer 3)** | on interaction | Signal stack (MACD, VWAP, OB imbalance, funding, consolidation rank) + position-size calculator (fixed % stop, R-multiple TPs, % of account risk). Outputs entry/stop/TPs, $ at risk, position notional, leverage, P&L per level. Deep-links to Hyperliquid's trade page. |
| 6 | **Macro reference** | 15s | Yahoo Finance prices for S&P, DXY, Gold, etc. — screening context, not tradeable here. |
| 7 | **AI Screening Summary** | 10s file-check | Displays `news_summary.md` written by Claude via the `/summarize-news` skill. |
| 8 | **News Headlines** | 60s | Yahoo Finance headlines per ticker. Also writes `news_dump.json` for Claude to consume. |

### Multi-DEX awareness

Hyperliquid is a *platform of perp DEXs* (HIP-3), not a single exchange. The dashboard's sidebar **"Browse Hyperliquid markets"** loads the full multi-DEX universe (~420 markets at time of writing — crypto, **commodities**, **equities**, **FX**, **indices**), classified and searchable. Typing `brentoil` auto-resolves to the canonical `xyz:BRENTOIL`.

## Quick start

```powershell
# 1. Get the code (copy the RIGA folder anywhere — no required path)
cd <wherever-you-put-it>\RIGA

# 2. Launch
.\run.ps1
```

That's it. The script will:
1. Create a `.venv` if missing
2. Install all Python dependencies from `requirements.txt`
3. Print local + LAN URLs
4. Launch Streamlit bound to `0.0.0.0:8501` (LAN-accessible)

Open **http://localhost:8501** in your browser. First load takes ~10s while the scanners fetch data.

### Linux / macOS

`run.ps1` is PowerShell. To run the dashboard manually on any OS:

```bash
python -m venv .venv
source .venv/bin/activate    # Windows: .venv\Scripts\activate
pip install -r requirements.txt
streamlit run app.py --server.address 0.0.0.0 --server.port 8501 --server.headless true
```

## LAN access — viewing from your phone / laptop

After `run.ps1`, devices on the same Wi-Fi can reach the dashboard at `http://<your-machine-lan-ip>:8501`. The launcher prints the URLs (e.g. `http://192.168.1.9:8501`).

**One-time firewall setup (Windows):**

```powershell
# Right-click PowerShell → "Run as Administrator", then:
cd <path>\RIGA
.\setup-firewall.ps1
```

Adds a `RIGA Dashboard` inbound rule for TCP 8501 (Private + Domain profiles — not Public, so coffee-shop Wi-Fi can't reach it). Remove later with `Remove-NetFirewallRule -DisplayName "RIGA Dashboard"`.

**Tailscale bonus:** if you have [Tailscale](https://tailscale.com) on this and another device (phone, laptop), you can hit the dashboard from anywhere over the Tailscale VPN at `http://<tailscale-ip>:8501` — no firewall changes, no port forwarding, no exposure to the public internet.

## Using the AI screening summary (Claude Code integration)

This is the killer feature for anyone with [Claude Code](https://claude.com/claude-code). The dashboard can't call an LLM API directly (your Claude.ai subscription has no programmatic API), so it uses a **file-based handshake**:

1. The dashboard auto-writes `news_dump.json` every 60s with the latest headlines from all your tracked tickers
2. In Claude Code (launched from the RIGA folder), invoke the slash command:
   ```
   /summarize-news
   ```
   Or just ask naturally: *"summarize today's news"*
3. Claude reads `news_dump.json`, writes a structured analysis to `news_summary.md` (themes, per-asset bias, Layer-2 candidates, cross-asset signals)
4. The dashboard's AI Summary panel auto-displays the markdown within 10 seconds

**How the slash command works:** `.claude/skills/summarize-news/SKILL.md` defines the protocol. Claude Code auto-loads project-level skills when launched from the directory.

No API key, no usage cost beyond your existing Claude subscription. You can re-run as often as you want.

## Configuring your watchlist

In the sidebar:

1. **Browse Hyperliquid markets** (expander) — filter by class (Crypto / Commodity / Equity / FX / Index) and DEX, then multi-select the assets you want to follow. Defaults to HYPE / ASTER / ZEC / TAO.
2. **Manual entries** — type any asset name (case-insensitive). The resolver finds it across all DEXs. E.g., `brentoil` → `xyz:BRENTOIL`, `aapl` → `xyz:AAPL`, `gold` → `xyz:GOLD`.
3. **Macro tickers** — Yahoo Finance symbols for screening context (not tradeable). E.g., `^GSPC` (S&P 500), `DX-Y.NYB` (DXY), `GC=F` (Gold futures), `BZ=F` (Brent), `CL=F` (WTI).

Selections persist for the current session.

### Code architecture

`app.py` is a single file divided into clear sections:

1. **Constants + classification helpers** — `DEFAULT_ASSETS`, asset-class name sets, `_classify_asset`
2. **Cached fetchers** — `get_hl_contexts`, `get_hl_book`, `get_candles_1h`, `get_yahoo_quote`, `get_yahoo_news`, `get_full_universe`, `get_context`, `resolve_asset`
3. **Signal computations** — `_consolidation_score`, `_macd_state`, `_vwap_state`, `_consolidation_chart`
4. **Streamlit fragments** (one per panel) — each panel is `@st.fragment(run_every="...")` so refresh rates are independent
5. **`main()`** — sidebar + panel orchestration

Refresh rates are managed by Streamlit's fragment API. The TradingView chart and Trade Decision panel are NOT fragments — they should only re-render on user interaction.

### Extending

- **New asset class on Hyperliquid?** Add the ticker name to one of `COMMODITY_NAMES`, `FX_NAMES`, `INDEX_NAMES` in `app.py`, and the classifier will pick it up.
- **Scanner across all DEXs?** The Top Movers and Consolidation Scanner currently use main DEX only. Replace the `get_hl_contexts()` call with `get_full_universe()` (returns 420+ assets) and iterate the resulting dict. Watch out for 5× the API load.
- **New signal in Trade Decision?** Add a computation function (look at `_macd_state` as a template) and a new column in the signal stack inside `trade_decision_panel()`.
- **New data source?** Add a `@st.cache_data` fetcher + a new fragment panel. Plug into `main()`.

## Known limitations

- **Scanners are main-DEX-only.** Top Movers and Consolidation Scanner don't see `xyz:BRENTOIL`-style cross-DEX markets yet. The per-asset panel and Trade Decision do.
- **Yahoo Finance** has thin or no coverage for many altcoins. The news panel and AI summary degrade gracefully (show "No news"), but expect Layer-1 to be light for HL-native tokens like HYPE / ASTER.
- **TradingView symbols don't auto-match Hyperliquid for non-crypto.** When you point the chart at `xyz:BRENTOIL`, you'll need to override the symbol manually (e.g. `TVC:UKOIL` or `OANDA:BCOUSD`). The chart panel suggests alternatives.
- **No persistence between sessions.** Sidebar settings, watchlist picks, trade plan inputs all reset when you close the browser tab. Use the Manual Entries field if you want a reliable default set.
- **No authentication.** Anyone on your LAN with the URL can view the dashboard. There's no login. This is fine for personal use — don't expose to the public internet without adding nginx + basic auth or similar.
- **Streamlit hot-reload caveat.** Code changes don't auto-rerun by default. Click "Rerun" in the top-right menu or press `R`.

## Troubleshooting

**"Address already in use" on port 8501** — another Streamlit instance is running. Find and kill it:
```powershell
Get-NetTCPConnection -LocalPort 8501 -State Listen | ForEach-Object { Stop-Process -Id $_.OwningProcess -Force }
```

**LAN URL doesn't work from another device** — run `setup-firewall.ps1` as admin (one-time).

**"Symbol doesn't exist" on TradingView chart** — change the venue prefix (`BYBIT:`, `MEXC:`, `KUCOIN:`, `COINBASE:`) or use the chart's built-in symbol search.

**Consolidation scanner is slow** — first scan takes ~10s because it fetches 1h candles for ~60 coins in parallel. Cached for 5min after that.

**Yahoo news returns "No news"** — Yahoo coverage is sparse for many crypto tickers, especially altcoins. Not a bug, just data scarcity.

**Universe browser shows 0 assets** — Hyperliquid API may be temporarily down. Try the "Force refresh (clear all caches)" button.
